FIX 2 — SCROLLABLE GRID (tiles hidden behind bottom bar)
════════════════════════════════════════════════════════

PROBLEM: The last row of category tiles on any page 
is partially or fully hidden behind the bottom 
navigation bar because the grid container does not 
account for the bar's height.

Fix the Athkar main screen (Screen 1) grid container:

STEP 1 — Import useSafeAreaInsets:
  import { useSafeAreaInsets } from 'react-native-safe-area-context'
  const insets = useSafeAreaInsets()

STEP 2 — Wrap the entire Screen 1 content 
(mode toggle + hint banner + page grid + dots indicator)
in a ScrollView:

  <ScrollView
    showsVerticalScrollIndicator={false}
    contentContainerStyle={{
      paddingBottom: insets.bottom + 80,
      // 80 = approximate bottom tab bar height
      // insets.bottom = device-specific safe area (notch phones)
    }}
  >
    {/* mode toggle */}
    {/* hint banner */}
    {/* page grid */}
    {/* dot indicators */}
  </ScrollView>

STEP 3 — The swipeable pages FlatList is HORIZONTAL
and must remain horizontal. It must NOT be inside 
a vertical ScrollView itself — instead, set the 
FlatList height explicitly:

  const GRID_ROWS = 4
  const GRID_HEIGHT = (TILE_HEIGHT * GRID_ROWS) + (GAP * (GRID_ROWS - 1))

  <FlatList
    horizontal
    pagingEnabled
    style={{ height: GRID_HEIGHT }}
    ...
  />

This gives the FlatList a fixed height equal to exactly
4 rows of tiles, then the outer ScrollView handles 
any overflow below it.

STEP 4 — Apply same paddingBottom fix to the 
Favourites page and the dhikr reader screen (Screen 2).

Do not change any category content, counts, or icons.
Do not add new dependencies — 
react-native-safe-area-context is already installed.
