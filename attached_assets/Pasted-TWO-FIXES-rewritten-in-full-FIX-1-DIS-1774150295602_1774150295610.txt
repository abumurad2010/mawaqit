FIX 1 — DISMISS BEHAVIOUR FOR ALL POPUPS APP-WIDE
════════════════════════════════════════════════════════

RULE: Tapping outside any popup, modal, sheet, overlay, 
or picker must ONLY close it. It must NEVER save, apply, 
or commit any change. Changes are only committed when 
the user explicitly taps a confirm/save/done button 
inside the popup.

STEP 2 — FOR EVERY POPUP FOUND, APPLY THIS PATTERN

Each popup must follow this exact state management pattern:

  // On open: snapshot current saved value into temp
  const handleOpen = () => {
    setTempValue(currentSavedValue)  // copy, don't reference
    setVisible(true)
  }

  // On dismiss (tap outside, back button, swipe away):
  const handleDismiss = () => {
    setTempValue(currentSavedValue)  // revert temp to saved
    setVisible(false)                // just close
    // DO NOT call any save/apply/update function here
  }

  // On explicit confirm (Done / Save / Apply / حفظ / تأكيد):
  const handleConfirm = () => {
    currentSavedValue = tempValue    // commit the change
    persistToStorage(tempValue)      // save if needed
    setVisible(false)                // then close
  }

  // All interactive elements inside the popup:
  // update tempValue only, never currentSavedValue directly

FIX 2 — ATHKAR TAB DEFAULT MODE
════════════════════════════════════════════════════════

RULE:
  The Athkar tab mode toggle has two states:
    'arabic'          — shows Arabic text only
    'transliterated'  — shows Arabic + transliteration 
                        + translation in selected language

DEFAULT ON EVERY LAUNCH / TAB VISIT:

  CASE A — User has never selected an app language
  (AsyncStorage language key is null or 'ar'):
    → Mode defaults to 'arabic'
    → Language selector hidden (arabic mode)
    → This is the out-of-box experience

  CASE B — User has selected a non-Arabic app language
  (AsyncStorage language key is 'en', 'fr', 'tr', etc.):
    → Mode defaults to 'transliterated'
    → Language selector defaults to the user's app language
    → Translation shown in that language

  In ALL cases, 'arabic' is the safe fallback.
  Never default to 'transliterated' unless a non-Arabic 
  app language is explicitly saved.

IMPLEMENTATION in app/(tabs)/athkar.tsx:

  // Get saved app language from context or AsyncStorage
  const { language } = useApp()  
  // or: const language = await AsyncStorage.getItem('appLanguage')

  // Determine default mode
  const defaultMode = (!language || language === 'ar')
    ? 'arabic'
    : 'transliterated'

  // Initialize mode state
  const [mode, setMode] = useState<'arabic' | 'transliterated'>(defaultMode)

  // Initialize language selector state
  const [athkarLang, setAthkarLang] = useState<string>(
    language || 'ar'
  )

  // When app language changes globally, update athkar defaults:
  useEffect(() => {
    if (!language || language === 'ar') {
      setMode('arabic')
      setAthkarLang('ar')
    } else {
      setMode('transliterated')
      setAthkarLang(language)
    }
  }, [language])

NOTE: Once the user manually toggles the mode or 
changes the language selector WITHIN the Athkar tab, 
their in-session choice takes over and must not be 
overridden until they leave and return to the tab.
On returning to the tab, re-apply the defaults above.
