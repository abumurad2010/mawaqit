FIX 2 — DEFAULT LANGUAGE: Arabic first, then follows 
         user's selected app language
════════════════════════════════════════════════════════

CURRENT PROBLEM: The app defaults to English or 
auto-detected language on first launch.

NEW BEHAVIOUR — apply app-wide across all tabs:

On FIRST LAUNCH (no language ever selected by user):
  → Default language: Arabic (ar)
  → All UI labels, prayer names, athkar categories,
    quran transliteration language, everything → Arabic

Once the USER EXPLICITLY SELECTS a language in Settings:
  → All tabs switch to that selected language
  → That selection persists via AsyncStorage 
    (already stored as existing language preference key)
  → Arabic is no longer the default from that point

IMPLEMENTATION:
In the language initialization logic 
(likely in contexts/AppContext.tsx or app/_layout.tsx),
find where the initial language is set.

Change the fallback/default from 'en' or 'auto' to 'ar':

  BEFORE (likely):
    const savedLang = await AsyncStorage.getItem('appLanguage')
    setLanguage(savedLang || 'en')
    // or: setLanguage(savedLang || deviceLocale || 'en')

  AFTER:
    const savedLang = await AsyncStorage.getItem('appLanguage')
    setLanguage(savedLang || 'ar')

This means:
  - If user has never set a language → Arabic
  - If user has set a language → that language
  - Auto-detection from device locale: DISABLED
    (remove any device locale fallback logic)

ALSO update the default in these specific places 
if they have their own language defaults:
  - Quran tab transliteration language selector → 'ar'
  - Athkar tab transliteration language selector → 'ar'
  - Any other tab with its own language state → 'ar'
